# 备忘录模式

## 思想

在不破坏封装的前提下，捕获对象内部状态并在**外部保存**，以便之后**恢复**到该状态（撤销/快照）。

## 结构

| 角色 | 说明 |
|------|------|
| Originator | 原发器，创建/恢复备忘录 |
| Memento | 存储 Originator 状态 |
| Caretaker | 保管备忘录，不访问内部细节 |

## 代码示例

```java
class Memento {
    private final String state;
    Memento(String state) { this.state = state; }
    String getState() { return state; } // 通常仅 Originator 可读
}

class Editor {
    private String content;
    Memento save() { return new Memento(content); }
    void restore(Memento m) { this.content = m.getState(); }
}
```

## 典型应用

- 编辑器 Undo/Redo
- 游戏存档
- 数据库事务 savepoint（思想类似）

## 深拷贝备忘录

状态含引用类型时，Memento 需**深拷贝**，否则恢复后仍共享可变对象。

## 面试一句话

> 备忘录保存对象状态快照供恢复，Caretaker 只存不读细节；编辑器撤销是经典场景。
