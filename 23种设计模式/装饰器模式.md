# 装饰器模式

## 思想

**动态**给对象添加职责，比继承更灵活；通过包装（组合）叠加功能，且可多层嵌套。

## 结构

- **Component**：抽象组件
- **ConcreteComponent**：被装饰的原对象
- **Decorator**：持有一个 Component 引用，实现同一接口，转发并增强

## 代码示例

```java
interface Coffee {
    double cost();
    String desc();
}

class SimpleCoffee implements Coffee {
    public double cost() { return 10; }
    public String desc() { return "coffee"; }
}

abstract class CoffeeDecorator implements Coffee {
    protected Coffee delegate;
    CoffeeDecorator(Coffee c) { this.delegate = c; }
}

class MilkDecorator extends CoffeeDecorator {
    MilkDecorator(Coffee c) { super(c); }
    public double cost() { return delegate.cost() + 2; }
    public String desc() { return delegate.desc() + ", milk"; }
}
```

## 典型应用（JDK）

| 类 | 说明 |
|----|------|
| `InputStream` 体系 | `BufferedInputStream`、`DataInputStream` 包装 |
| `Reader` / `Writer` | 层层装饰 |
| `Collections.synchronizedList` | 装饰 List 加同步 |

## 装饰 vs 代理

| | 装饰器 | 代理 |
|---|--------|------|
| 目的 | **增强功能**（可多层） | **控制访问** |
| 对客户端 | 通常透明叠加 | 可能代表真实对象 |
| 创建 | 客户端主动包装 | 常由框架生成 |

## 面试一句话

> 装饰器用包装类动态叠加功能；Java IO 流是标准范例，与代理的区别在「功能增强」vs「访问控制」。
