# 组合模式

## 思想

将对象组合成**树形结构**以表示「部分-整体」层次，使客户端对**单个对象和组合对象**的使用具有一致性。

## 适用场景

- 树形结构：菜单、文件夹、组织架构、UI 组件树
- 需要统一对待叶子节点和容器节点

## 结构

| 角色 | 说明 |
|------|------|
| Component | 抽象节点，声明公共操作 |
| Leaf | 叶子，无子节点 |
| Composite | 容器，持有子 Component 列表 |

## 代码示例

```java
interface MenuComponent {
    void print();
}

class MenuItem implements MenuComponent {
    private String name;
    public void print() { System.out.println("  " + name); }
}

class Menu implements MenuComponent {
    private List<MenuComponent> children = new ArrayList<>();
    public void add(MenuComponent c) { children.add(c); }
    public void print() {
        children.forEach(MenuComponent::print);
    }
}
```

## 透明式 vs 安全式

| 方式 | 特点 |
|------|------|
| 透明式 | Component 统一有 add/remove，Leaf 抛异常或空实现 |
| 安全式 | 只有 Composite 有 add/remove，Leaf 无这些方法 |

## 典型应用

- Java AWT/Swing `Component` 树
- MyBatis 动态 SQL 节点组合
- 文件系统目录结构

## 面试一句话

> 组合模式让树形结构中叶子和容器对外接口一致，统一递归处理；文件夹+文件是经典例子。
