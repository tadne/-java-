# 状态模式

## 思想

允许对象在**内部状态改变**时改变其行为，对象看起来好像修改了它的类（把状态相关行为拆到不同状态类）。

## 结构

- **Context**：持有当前 State，委托行为给 State
- **State**：抽象状态，声明行为接口
- **ConcreteState**：各状态下不同实现

## 代码示例

```java
interface State {
    void handle(Order context);
}

class PaidState implements State {
    public void handle(Order ctx) {
        ctx.setState(new ShippedState());
        // 发货逻辑
    }
}

class Order {
    private State state = new UnpaidState();
    void setState(State s) { this.state = s; }
    void next() { state.handle(this); }
}
```

## 与策略模式

| | 状态 | 策略 |
|---|------|------|
| 切换 | **对象内部**自动切换 | **客户端**主动指定 |
| 关系 | 状态间可能知道彼此 | 策略间通常独立 |
| 场景 | 订单状态机、TCP 连接状态 | 排序算法、支付方式 |

## 典型应用

- 工作流引擎状态节点
- 线程 `BLOCKED` / `RUNNABLE` 等（概念类似）
- 游戏角色状态（站立/跑步/攻击）

## 面试一句话

> 状态模式把行为拆到状态类，Context 委托给当前 State；与策略的区别是状态**自动流转**。
