# 对象一定分配在堆中吗

**不一定。** 规范层面对象在堆上；JVM 可通过**逃逸分析**做优化，使部分对象不占用堆。

## 逃逸分析

| 逃逸程度 | 优化可能 |
|----------|----------|
| 未逃逸 | 栈上分配、标量替换、锁消除 |
| 方法逃逸 | 优化受限 |
| 线程逃逸 | 必须堆上，线程间可见 |

## 常见优化

| 优化 | 说明 |
|------|------|
| **栈上分配** | 对象未逃出方法，在栈帧销毁时一并回收 |
| **标量替换** | 不建对象，拆成局部变量 |
| **锁消除** | 仅单线程使用的同步 |

## 开启（了解）

- `-XX:+DoEscapeAnalysis`（默认开启，以 JDK 为准）

## 面试一句话

> 语法上 new 在堆；JVM 对未逃逸对象可栈上分配或标量替换，故“对象一定在堆上”不严谨。
