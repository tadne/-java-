# 元空间代替永久代

## 永久代（PermGen，JDK 7 及以前）

- 位于 **Java 堆** 中的特殊区域
- 存储：类元数据、运行时常量池（部分）、静态变量等
- 大小固定：`-XX:PermSize`、`-XX:MaxPermSize`（默认约 64MB～）
- 问题：易 **PermGen OOM**、Full GC 效率低、调优困难

## 元空间（Metaspace，JDK 8+）

- 使用 **本地内存（Native Memory）**，不再占用堆中的 PermGen
- 主要存储：**类元数据**（Klass、方法元信息等）
- 默认上限很大，可用 `-XX:MaxMetaspaceSize` 限制

```bash
-XX:MetaspaceSize=128m
-XX:MaxMetaspaceSize=256m
```

## 为什么要替换

| 对比 | 永久代 | 元空间 |
|------|--------|--------|
| 内存来源 | 堆内，受堆大小限制 | 本地内存，与堆分离 |
| 扩容 | 固定，易 OOM | 可动态扩展（受 Max 限制） |
| 字符串常量 | 部分在 PermGen | **JDK7** 起字符串常量池移至堆；**JDK8** 彻底移除 PermGen |
| 规范 | HotSpot 特有实现 | 更符合规范化的方法区实现思路 |

> **易错**：不是「永久代不符合 JVM 规范」这么简单，而是 HotSpot 将方法区实现从堆内 PermGen 改为本地 Metaspace，降低 OOM 与 GC 压力。

## 类卸载与 GC

- 元空间不足会触发 **Full GC**（或 G1 等收集器的并发周期）尝试**卸载无用类**
- 类卸载条件苛刻：ClassLoader 可回收、该类所有实例已不存在等
- 动态代理、反射、OSGi 等大量生成类场景需关注 Metaspace

## 面试一句话

> JDK8 用元空间（本地内存）替代堆内永久代，主要存类元数据；字符串常量池在堆上；通过 MaxMetaspaceSize 防无限增长。
