# 访问者模式

## 思想

表示一个作用于某对象结构中各元素的操作，它使你可以在不改变各元素类的前提下**定义作用于这些元素的新操作**（把操作从数据结构中分离）。

## 结构

- **Visitor**：为每种 ConcreteElement 声明 `visit(XxxElement)`
- **Element**：接受 Visitor（`accept(Visitor v)`）
- **ObjectStructure**：元素集合

## 代码示例

```java
interface Visitor {
    void visit(Book b);
    void visit(Fruit f);
}

interface Element {
    void accept(Visitor v);
}

class Book implements Element {
    public void accept(Visitor v) { v.visit(this); }
}

class PriceVisitor implements Visitor {
    public void visit(Book b) { /* 计算书价 */ }
    public void visit(Fruit f) { /* 计算水果价 */ }
}
```

## 双重分派

`element.accept(visitor)` 内部再调 `visitor.visit(this)`，实现**运行时**确定元素类型与访问操作。

## 典型应用

- 编译器 AST 遍历（不同 Visitor 做类型检查、代码生成）
- ASM 字节码访问
- 复杂对象结构上的多种统计/导出操作

## 缺点

- 新增 **Element 类型**要改所有 Visitor（违反开闭）
- 新增 **操作**只需加新 Visitor（符合开闭）

## 面试一句话

> 访问者把操作抽到 Visitor，Element 用 accept 接受；适合结构稳定、操作常增的场景，如 AST、ASM。
