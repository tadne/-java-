# Java 的自动装箱与拆箱

## 概念

- **装箱**：基本类型 → 包装类（如 `int` → `Integer`）
- **拆箱**：包装类 → 基本类型

编译器自动插入 `valueOf` / `xxxValue()`。

## 注意

| 问题 | 说明 |
|------|------|
| NPE | 拆箱时包装类为 null |
| 性能 | 循环中频繁装箱有开销 |
| 比较 | `Integer` 用 `equals`，`==` 对 -128~127 可能为 true（缓存） |

## 面试一句话

> 装箱拆箱是语法糖；注意 null 拆箱 NPE 与 Integer 缓存导致的 == 陷阱。
