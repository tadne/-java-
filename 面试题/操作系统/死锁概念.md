# 死锁概念

## 定义

多个进程/线程因**竞争不可抢占资源**且**互相等待**对方释放，导致都无法继续执行。

## 四个必要条件（同时满足才可能死锁）

| 条件 | 说明 |
|------|------|
| 互斥 | 资源一次只能被一个占用 |
| 占有且等待 | 已占资源同时等待其他资源 |
| 不可抢占 | 只能由持有者主动释放 |
| 循环等待 | 存在资源等待环 |

## 处理策略

| 策略 | 做法 |
|------|------|
| 预防 | 破坏四条件之一（如固定加锁顺序） |
| 避免 | 银行家算法（分配前安全性检查） |
| 检测与恢复 | 检测等待图环，杀死或回滚 |
| 鸵鸟 | 不处理，依赖重启 |

## 面试一句话

> 死锁是互相等待导致的僵局；需四条件同时具备，常通过固定加锁顺序或 tryLock 超时避免。
