# 代理模式

## 思想

为对象提供一个**代理**，由代理控制对真实对象的访问，可在调用前后增加逻辑（日志、权限、事务、延迟加载等）。

## 结构

- **Subject**：抽象主题（接口）
- **RealSubject**：真实对象
- **Proxy**：实现同一接口，持有 RealSubject，转发并增强

## 三种实现方式

| 方式 | 机制 | 特点 |
|------|------|------|
| **静态代理** | 手写代理类实现接口 | 简单；每个被代理类要一个代理类 |
| **JDK 动态代理** | `Proxy.newProxyInstance` + `InvocationHandler` | 只能代理**接口** |
| **CGLIB 代理** | 子类继承被代理类 | 可代理类（无接口）；不能代理 `final` |

## JDK 动态代理要点

```java
Subject proxy = (Subject) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),
    target.getClass().getInterfaces(),
    (proxy, method, args) -> {
        // before
        Object ret = method.invoke(target, args);
        // after
        return ret;
    });
```

## 典型应用

- Spring AOP、RPC 客户端桩、MyBatis Mapper 接口

## 面试要点

- 目的：控制访问 + 增强，不改原类代码
- JDK 代理基于接口；CGLIB 基于子类
- 与装饰器模式：装饰器侧重功能叠加；代理侧重访问控制
