# 常用线程池

## 为什么用线程池

- 频繁创建/销毁线程开销大
- 线程池**复用线程**、控制并发数、管理任务队列

## 核心参数（ThreadPoolExecutor）

| 参数 | 含义 |
|------|------|
| corePoolSize | 核心线程数 |
| maximumPoolSize | 最大线程数 |
| keepAliveTime | 非核心线程空闲存活时间 |
| workQueue | 任务队列 |
| threadFactory | 线程工厂 |
| handler | 拒绝策略 |

## execute 与 submit

| | execute | submit |
|---|---------|--------|
| 来源 | Executor | ExecutorService |
| 参数 | Runnable | Runnable / Callable |
| 返回值 | void | `Future` |
| 异常 | Runnable 异常直接抛出 | 异常封装在 Future，get 时抛出 |

## Executors 工厂方法（了解 + 坑）

| 方法 | 特点 | 风险 |
|------|------|------|
| `newFixedThreadPool(n)` | 固定 n 线程，**无界队列** | 队列堆积 OOM |
| `newCachedThreadPool()` | 可无限扩线程 | 高并发创建大量线程 |
| `newSingleThreadExecutor()` | 单线程顺序执行 | 无界队列 OOM |
| `newScheduledThreadPool(n)` | 定时/周期任务 | 无界队列 |

> **阿里巴巴规范**：不推荐用 Executors 创建，应显式 `new ThreadPoolExecutor(...)` 并指定**有界队列**和拒绝策略。

## 推荐创建方式

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    4,                      // core
    8,                      // max
    60L, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),
    Executors.defaultThreadFactory(),
    new ThreadPoolExecutor.CallerRunsPolicy()
);
```

## 拒绝策略

| 策略 | 行为 |
|------|------|
| AbortPolicy | 抛 RejectedExecutionException（默认） |
| CallerRunsPolicy | 调用者线程执行 |
| DiscardPolicy | 静默丢弃 |
| DiscardOldestPolicy | 丢弃队列最老任务 |

## 线程数设置思路

- CPU 密集：`N + 1`（N = CPU 核数）
- IO 密集：`2N` 或 `N / (1 - 阻塞系数)`，需压测
- 业务线程池与 CPU 线程池分离

## 面试一句话

> 用 ThreadPoolExecutor 明确核心/最大线程、有界队列和拒绝策略；慎用 Executors 无界队列工厂方法。
