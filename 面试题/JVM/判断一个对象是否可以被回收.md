# 判断一个对象是否可以被回收

## 可达性分析（主流）

从 **GC Roots** 出发，不可达的对象可被回收（并非立即回收）。

### 常见 GC Roots

| 类型 | 示例 |
|------|------|
| 线程栈局部变量 | 栈帧中的引用 |
| 静态变量 | 方法区/元空间中类静态字段 |
| 常量 | 运行时常量池引用 |
| JNI 引用 | Native 方法引用的对象 |
| 同步锁 | 持有 Monitor 的对象 |
| JVM 内部引用 | 系统类加载器等 |

## 两次标记（finalize）

1. 第一次标记：不可达；
2. 若覆写 `finalize()` 且未执行过，放入 F-Queue 一次机会复活；
3. 第二次标记仍不可达则真正回收。

> **不推荐**依赖 `finalize`，JDK 9+ 已废弃相关机制倾向。

## 引用类型（补充）

| 引用 | 回收时机 |
|------|----------|
| 强 | 永不因引用类型本身回收 |
| 软 | OOM 前清理 |
| 弱 | 下次 GC |
| 虚 | 无法通过虚引用取对象，用于堆外清理跟踪 |

## 面试一句话

> 以 GC Roots 可达性为准；finalize 仅一次机会且不可靠，实际开发避免依赖。
