# 工厂模式

工厂模式解决**谁创建对象、创建哪种对象**的问题，包含三种常见形态。

## 简单工厂（不属于 GoF 23，但常一起讲）

一个工厂类根据参数 `if/switch` 创建产品。

```java
public interface Shape { void draw(); }

public class Circle implements Shape {
    public void draw() { System.out.println("circle"); }
}

public class ShapeFactory {
    public static Shape create(String type) {
        return switch (type) {
            case "circle" -> new Circle();
            default -> throw new IllegalArgumentException(type);
        };
    }
}
```

**缺点**：新增产品要改工厂类，违反开闭原则（OCP）。

---

## 工厂方法（Factory Method）

定义创建产品的接口，**子类决定**实例化哪个具体类。

```java
interface Product { void use(); }

class ProductA implements Product {
    public void use() { System.out.println("A"); }
}

abstract class Factory {
    abstract Product create();
    void doSomething() {
        Product p = create();
        p.use();
    }
}

class FactoryA extends Factory {
    Product create() { return new ProductA(); }
}
```

**优点**：新增产品 = 新增工厂子类，符合 OCP。

---

## 抽象工厂（Abstract Factory）

提供一个接口，用于创建**一族**相关或依赖的对象，而不指定具体类。

```java
interface Button { void render(); }
interface Checkbox { void check(); }

interface GUIFactory {
    Button createButton();
    Checkbox createCheckbox();
}

class WinFactory implements GUIFactory {
    public Button createButton() { return new WinButton(); }
    public Checkbox createCheckbox() { return new WinCheckbox(); }
}

class MacFactory implements GUIFactory {
    public Button createButton() { return new MacButton(); }
    public Checkbox createCheckbox() { return new MacCheckbox(); }
}
```

**优点**：保证产品族内风格一致（全 Win 或全 Mac UI）。  
**缺点**：扩展新产品族要改抽象工厂接口。

---

## 三种对比

| 类型 | 创建粒度 | 扩展方式 |
|------|----------|----------|
| 简单工厂 | 一种产品，参数区分 | 改工厂类 |
| 工厂方法 | 一种产品 | 加工厂子类 |
| 抽象工厂 | **一族**产品 | 加工厂实现类 |

## 典型应用

- Spring `BeanFactory` / `@Bean`：容器负责创建与管理 Bean
- `Calendar.getInstance()`：简单工厂思想
- JDBC `DriverManager.getConnection()`：按驱动创建连接

## 面试一句话

> 简单工厂一个类 switch 创建；工厂方法子类决定产品；抽象工厂创建一族相关产品；Spring Bean 创建是工厂思想的应用。
