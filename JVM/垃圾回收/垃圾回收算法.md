# 垃圾回收算法

## 1. 标记-清除（Mark-Sweep）

- **标记**：对 GC Root 根对象所引用的对象进行标记
- **清除**：对没有标记的对象的起始和结束地址进行记录，放入空闲地址列表，然后释放其内存
- **优点**：速度快
- **缺点**：会产生内存碎片

## 2. 标记-整理（Mark-Compact）

- **标记**：对 GC Root 根对象所引用的对象进行标记
- **整理**：将没有被标记的对象释放，然后对被标记对象的起始和结束地址进行整理，使内存更紧凑
- **优点**：没有内存碎片
- **缺点**：速度慢（需要移动对象）

## 3. 复制（Copying）

- 将内存区域划分为 `from` 和 `to` 两个部分，`to` 是一个空区域
- 先标记，找到被引用的对象，将这些对象从 `from` 区域复制到 `to` 区域
- 然后删除 `from` 区域中所有内存
- 最后交换 `from` 和 `to` 区域的位置
- **优点**：不会产生碎片
- **缺点**：占用双倍内存空间

## 三种算法对比

| 算法 | 速度 | 内存碎片 | 空间利用率 |
|------|------|----------|------------|
| 标记-清除 | 快 | 有 | 高 |
| 标记-整理 | 慢 | 无 | 高 |
| 复制 | 快 | 无 | 低（需要双倍空间） |

## 面试要点

- **标记-清除**：速度快，但会产生内存碎片
- **标记-整理**：无内存碎片，但速度慢（需移动对象）
- **复制**：无碎片且速度快，但需要双倍内存空间
- 实际 JVM 中通常采用**分代收集**，结合多种算法：新生代用复制，老年代用标记-清除或标记-整理
