# 观察者模式

## 思想

定义对象间**一对多**依赖，当被观察者状态改变时，**所有观察者**自动收到通知并更新。

## 结构

- **Subject（Observable）**：维护观察者列表，状态变化时 `notify`
- **Observer**：定义更新接口
- **ConcreteSubject / ConcreteObserver**

## 代码示例

```java
interface Observer {
    void update(String msg);
}

class Subject {
    private List<Observer> observers = new ArrayList<>();
    void attach(Observer o) { observers.add(o); }
    void notifyAll(String msg) {
        observers.forEach(o -> o.update(msg));
    }
}
```

## JDK 与 Spring

| 实现 | 说明 |
|------|------|
| `java.util.Observable` | 已过时，不推荐 |
| 事件驱动 | `ApplicationEvent` + `@EventListener` |
| MQ | 发布/订阅是分布式观察者 |
| React/Vue | 响应式数据绑定 |

## 推模型 vs 拉模型

- **推**：Subject 把详细数据传给 Observer
- **拉**：Observer 收到通知后自己从 Subject 拉取需要的数据

## 面试一句话

> 观察者是一对多通知；Spring 事件、MQ 发布订阅都是其延伸，JDK Observable 已废弃。
