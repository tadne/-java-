# 中介者模式

## 思想

用一个**中介对象**封装一系列对象之间的交互，使各对象不必显式相互引用，降低**网状耦合**。

## 结构

- **Mediator**：定义同事对象通信接口
- **ConcreteMediator**：协调各 Colleague
- **Colleague**：只与 Mediator 通信，不直接引用其他 Colleague

## 代码示例

```java
interface Mediator {
    void notify(Colleague sender, String event);
}

class ChatRoom implements Mediator {
    public void notify(Colleague sender, String event) {
        // 转发消息给其他用户
    }
}

class User extends Colleague {
    void send(String msg) { mediator.notify(this, msg); }
}
```

## 典型应用

- MVC 中 **Controller** 协调 Model 与 View
- 消息中间件（MQ）解耦生产者与消费者
- 机场塔台调度飞机

## 与观察者

| | 中介者 | 观察者 |
|---|--------|--------|
| 通信 | **多向**，经中心协调 | **一对多**，Subject 通知 Observer |
| 耦合 | Colleague 只依赖 Mediator | Observer 订阅 Subject |

## 面试一句话

> 中介者用中心对象协调多方交互，避免 N×N 引用；聊天室、MQ、MVC Controller 都是变体。
