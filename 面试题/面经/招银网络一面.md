# 招银网络一面

## 四种访问修饰符

| 修饰符 | 访问范围 |
|--------|----------|
| **public** | 任意包下的类 |
| **protected** | 同一包内 + 不同包的子类 |
| **default**（缺省） | 同一包下的类 |
| **private** | 仅本类 |

## 四种非访问修饰符

| 修饰符 | 作用 |
|--------|------|
| **static** | 成员变量、方法属于类而不是实例 |
| **final** | 表示最终变量、方法，不可修改 |
| **abstract** | 表示类或方法不能被实例化 |
| **synchronized** | 确保同一时间只有一个线程可以访问当前类或方法 |
| **volatile** | 保证多线程下的可见性 |

## 重载和重写

| 特性 | 重载（Overload） | 重写（Override） |
|------|------------------|------------------|
| 位置 | 同一个类中 | 子类或实现类中 |
| 方法名 | 相同 | 相同 |
| 参数 | 不同 | 相同 |
| 返回类型 | 可以不同 | 相同或协变 |
| 目的 | 实现多态 | 实现多态 |

## ArrayList 和 LinkedList 的区别

| 特性 | ArrayList | LinkedList |
|------|-----------|------------|
| 底层结构 | 基于数组实现 | 基于链表实现 |
| 索引访问 | 支持，随机访问性能好 | 不支持，随机访问性能差 |
| 增删操作 | 效果差（需要移动元素） | 效果好（只需修改指针） |

## 多态理解

- 多态主要是一个东西的多种形态，有继承多态和接口多态。
- 主要就是子类或者实现类可以作为参数构建父类，当调用时，使用的是父类方法，但是实际实现是在子类中。

### 项目中的应用

- 多态的应用就比较普遍了，比如创建 List 集合的时候，再比如三层架构调用 service 层接口的时候。

### 好处

- **使用灵活**：父类操作子类适应不同类型
- **方便扩展**：使用时继承父类即可
- **可维护**：父类引用子类对象，修改时只需要修改父类的引用即可
- **代码复用**：在父类下定义子类的实现

## TCP 三次握手

| 步骤 | 动作 | 说明 |
|------|------|------|
| 第一次 | 客户端发送 SYN | 发起连接 |
| 第二次 | 服务端发送 SYN+ACK | 确认连接并发起连接 |
| 第三次 | 客户端发送 ACK | 确认连接 |

### 为什么要三次握手？

- 三次握手的首要原因是为了防止旧的重复连接初始化造成混乱。
- 如果客户端发送 SYN 后，客户端出了问题，此时消息被网络阻塞了。那么当客户端重启后，再次建立连接，这个时候服务端又收到之前的消息，就会导致混乱。
- 三次握手通过客户端的确认号、服务端的确认号来确认连接的唯一性。

## TCP 序列号、确认号

| 概念 | 作用 |
|------|------|
| **序列号** | 标识 TCP 数据流中数据段的顺序和位置；也用于流量控制和拥塞控制，帮助调整发送速率，防止丢包和网络阻塞 |
| **确认号** | 告知对方确认已经成功接收数据段，并且保证连接唯一性；实现 TCP 可靠性，根据确认号进行超时重传 |

## 键入网址到网页显示流程

1. 浏览器解析 URL
2. DNS 根据域名解析出 IP 地址
3. 协议栈负责发送传输数据
   - 同一主机下：TCP 包头，封装源端口和目标端口，加上窗口大小、确认号等
   - 主机与路由器：IP 包头，封装源 IP 和目标 IP
   - 不同主机间：MAC 包头，封装源 MAC 和目标 MAC，其中目标 MAC 可变
4. 网卡负责将数字信息转换为电信号
5. 交换机根据 MAC 地址表查 MAC 地址，然后将信号发到路由器
6. 路由器接收发送来的网络包，然后通过路由表转发目标网络的路由器
7. 目标网络路由器转发给目标主机

> **IP 是全局范围内唯一标识，MAC 地址是局域网内唯一标识。**

## HTTP 和 TCP 长连接的区别

| 特性 | HTTP | TCP |
|------|------|-----|
| 层级 | 应用层协议 | 传输层协议 |
| 作用 | 控制在客户端和服务器之间传输数据 | 负责提供可靠的数据传输服务 |
| 长连接 | HTTP 可以利用 TCP 长连接来提高性能，减少连接建立和断开的开销 | TCP 长连接是在 TCP 层的概念，允许在一段时间内保持 TCP 连接的打开状态 |

### HTTP 版本演进

| 版本 | 协议 | 特性 |
|------|------|------|
| HTTP/1 | TCP | 采用长连接优化 |
| HTTP/2 | TCP | 采用 Stream 并发传输，多路复用 |
| HTTP/3 | QUIC（基于 UDP） | 使用 UDP 实现 |

## 接口和抽象类的区别

| 特性 | 接口（interface） | 抽象类（abstract class） |
|------|------------------|-------------------------|
| 关键字 | `interface` | `abstract` |
| 方法实现 | 只能包含方法声明（JDK 8 前） | 可以包含具体方法实现和抽象方法声明 |
| 构造函数 | 不能有 | 可以有 |
| 成员变量 | 只能有常量 | 可以有成员变量 |
| 实现机制 | 通过实现（implements） | 通过继承（extends） |

### 使用场景

- 如果多个类需要共享某些行为但没有共同的实现，可使用**接口**
- 如果多个类有共同的实现或者需要包含一些状态和行为，可以使用**抽象类**

### 例子

- 集合中，顶层接口 `Collection`，子接口 `List`、`Set`、`Map` 等
- Spring 中，`AbstractApplicationContext` 是一个抽象类，提供通用的上下文功能、启动关闭上下文的模板方法，提供扩展、管理生命周期

## 进程和线程

| 特性 | 进程 | 线程 |
|------|------|------|
| 定位 | 资源分配的基本单位 | 操作系统调度的基本单位 |
| 独立性 | 进程间相互独立 | 线程间共享相同的内存空间和资源 |
| 切换开销 | 大 | 小（共享地址空间） |

## 进程间的通信

| 方式 | 说明 |
|------|------|
| **管道** | 半双工通信，用于父子进程 |
| **消息队列** | 每个消息都有标识符和数据，进程通过标识符进行消息发送和接收 |
| **信号量** | 控制对共享资源的访问 |
| **信号** | 通知进程发生事件的机制 |

## 信号使用

1. 注册信号处理器，一般使用 `signal` 函数或者 `sigaction` 函数注册
2. 定义信号处理器函数，处理接收的特定信号
3. 发送信号，通过命令行或者其他进程或操作系统发送
4. 处理信号，进程会调用事先注册的信号处理器函数

## IO 多路复用

### 主要的 IO 复用技术

| 技术 | 说明 | 限制 |
|------|------|------|
| **select()** | 系统调用，允许程序同时监视多个文件描述符的读写状态 | 监视的文件描述符数量有限，一般为 1024 |
| **poll()** | 系统调用，没有 select 数量限制 | 监视一个文件描述符数组，当文件描述符准备好读或写时，poll 将返回 |
| **epoll()** | Linux 特有，大规模并发连接时，性能最好 | 事件驱动，效率高；使用三个系统调用：创建 epoll 实例、控制文件描述符添加和删除、等待事件 |

### 优势

- 减少系统调用次数、提高并发、降低 CPU 使用率
- 适用于管理多个 IO 流的场景，如网络服务器、聊天应用等

## 如何提高线程效率

1. **减少线程的数量**：避免创建过多的线程，线程的创建和切换会消耗系统资源。考虑使用线程池来管理线程，重复利用已有的线程。
2. **避免线程阻塞**：避免线程在执行过程中被阻塞，使用非阻塞的 I/O 操作、异步操作或使用多线程和多进程结合的方式处理任务。
3. **合理使用同步机制**：在多线程环境中，需要合理使用锁、条件变量等同步机制，避免出现死锁和竞态条件。
4. **注意线程间通信**：线程间通信需要谨慎处理，选择合适的通信方式（如队列、信号量等），避免频繁的线程间切换和数据拷贝。
5. **优化算法和数据结构**：在多线程编程中，选择合适的算法和数据结构可以减少线程间的竞争和冲突，提高程序的并发性能。
6. **使用并发工具和框架**：使用现有的并发工具和框架可以简化多线程编程的复杂性，提高开发效率和性能。
7. **合理分配任务**：将任务合理地分配给不同的线程，避免某个线程负载过重，导致其他线程处于空闲状态。
8. **注意资源管理**：合理管理线程所需的资源，及时释放不再需要的资源，避免资源泄露和浪费。
