# TCP 三次握手

建立连接前，客户端与服务器需交换初始序列号并确认双方收发能力正常。

## 流程表

| 次序 | 发送方 | 标志 | 关键字段 | 状态 |
|------|--------|------|----------|------|
| 1 | Client | SYN=1 | seq=x | Client: SYN_SENT |
| 2 | Server | SYN=1, ACK=1 | seq=y, ack=x+1 | Server: SYN_RCVD |
| 3 | Client | ACK=1 | seq=x+1, ack=y+1 | 双方: ESTABLISHED |

## 各步含义

1. **第一次**：客户端请求建立连接，携带初始序号 `seq=x`
2. **第二次**：服务端确认客户端 SYN，并带上自己的 `seq=y`，`ack=x+1`
3. **第三次**：客户端确认服务端 SYN，`ack=y+1`，连接建立

## 字段说明

| 字段 | 含义 |
|------|------|
| seq | 本报文数据起始序号 |
| ack | 期望对方下一个字节的序号（确认号） |
| SYN | 同步序号，用于建连 |
| ACK | 确认有效 |

## 为何三次

- 两次无法可靠确认客户端的接收能力
- 防止已失效的连接请求被误当作新连接（历史 SYN）

## 面试一句话

> SYN → SYN+ACK → ACK 三步，交换并确认 seq/ack 后进入 ESTABLISHED。
