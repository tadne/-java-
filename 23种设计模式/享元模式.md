# 享元模式

## 思想

运用**共享**技术有效支持大量**细粒度**对象，将可共享的状态（内部状态）与不可共享的状态（外部状态）分离。

## 核心概念

| 概念 | 说明 |
|------|------|
| 内部状态（intrinsic） | 可共享、与上下文无关 |
| 外部状态（extrinsic） | 不可共享、由客户端传入 |
| 享元工厂 | 缓存已创建的享元，避免重复 |

## 代码示例

```java
class ChessPiece { // 享元
    private String color; // 内部状态
    ChessPiece(String color) { this.color = color; }
    void display(int x, int y) { // x,y 外部状态
        System.out.println(color + " at " + x + "," + y);
    }
}

class FlyweightFactory {
    private static Map<String, ChessPiece> pool = new HashMap<>();
    static ChessPiece get(String color) {
        return pool.computeIfAbsent(color, ChessPiece::new);
    }
}
```

## 典型应用

- `Integer.valueOf(-128~127)` 缓存（`-XX:AutoBoxCacheMax`）
- `String` 常量池
- 线程池、连接池（广义共享资源思想）

## 注意

- 外部状态由调用方传入，享元本身无该状态
- 工厂 + Map 缓存是标准写法
- 对象过多时才值得用，否则增加复杂度

## 面试一句话

> 享元分离内部/外部状态并复用内部状态；Integer 缓存、String 池是 JDK 中的体现。
