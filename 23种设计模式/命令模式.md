# 命令模式

## 思想

将**请求封装成对象**，从而用不同的请求、队列、日志来参数化客户端，并支持**撤销/重做**。

## 结构

| 角色 | 说明 |
|------|------|
| Command | 命令接口，`execute()` / `undo()` |
| ConcreteCommand | 绑定 Receiver 与动作 |
| Invoker | 调用命令，不关心具体执行 |
| Receiver | 真正干活的类 |

## 代码示例

```java
interface Command {
    void execute();
    void undo();
}

class LightOnCommand implements Command {
    private Light light;
    LightOnCommand(Light light) { this.light = light; }
    public void execute() { light.on(); }
    public void undo() { light.off(); }
}

class RemoteControl {
    private Command command;
    void setCommand(Command c) { this.command = c; }
    void press() { command.execute(); }
}
```

## 典型应用

- `Runnable` / `Callable`：命令对象 + 线程执行
- 事务回滚、编辑器 Undo/Redo
- 消息队列：消息体即命令对象

## 面试一句话

> 命令把操作封装成对象，解耦调用者与执行者，便于队列、日志和撤销；Runnable 是简化版命令。
