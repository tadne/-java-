# 建造者模式

## 思想

将复杂对象的**构建步骤**与**表示**分离，使同样的构建过程可以创建不同的表示。

## 适用场景

- 构造参数多、可选参数多（避免 telescoping constructor）
- 需要保证构建过程有序（如先设必填再设可选）
- 构建结果不可变（构建完返回不可变对象）

## 结构

| 角色 | 说明 |
|------|------|
| Product | 最终产品 |
| Builder | 定义构建步骤接口 |
| ConcreteBuilder | 实现各步骤，产出 Product |
| Director（可选） | 编排构建顺序 |

## 代码示例

```java
public class User {
    private final String name;
    private final int age;
    private User(Builder b) {
        this.name = b.name;
        this.age = b.age;
    }
    public static class Builder {
        private String name;
        private int age;
        public Builder name(String name) { this.name = name; return this; }
        public Builder age(int age) { this.age = age; return this; }
        public User build() { return new User(this); }
    }
}

// 使用
User user = new User.Builder().name("Tom").age(20).build();
```

## 典型应用

- `StringBuilder` / `StringBuffer`
- Lombok `@Builder`
- MyBatis `SqlSessionFactoryBuilder`
- `HttpClient` 构建器

## 与工厂模式

| | 建造者 | 工厂 |
|---|--------|------|
| 关注 | **如何一步步组装** | **创建哪个具体类** |
| 产品 | 通常复杂、多字段 | 相对简单 |

## 面试一句话

> 建造者把复杂对象拆成链式/分步构建，避免构造器参数爆炸；JDK 中 StringBuilder、Lombok Builder 都是典型。
