# 单例模式

## 思想

保证一个类**只有一个实例**，并提供**全局访问点**。

## 实现要点

- **私有构造方法**，禁止外部 `new`
- 通过 **静态字段/方法** 或 **枚举** 暴露唯一实例
- 多线程场景注意**可见性与只创建一次**

## 常见写法

| 方式 | 特点 |
|------|------|
| 饿汉式 | 类加载即创建，简单，可能浪费 |
| 懒汉式 + DCL | 延迟加载，需 `volatile` + 双重检查 |
| 静态内部类 | 延迟加载，利用类加载保证线程安全 |
| 枚举 | 防反射/序列化破坏，不能延迟加载 |

## 代码示例

**饿汉式**

```java
public class Singleton1 {
    private static final Singleton1 INSTANCE = new Singleton1();
    private Singleton1() {}
    public static Singleton1 getInstance() { return INSTANCE; }
}
```

**懒汉式（双重检查锁定）**

```java
public class Singleton2 {
    private static volatile Singleton2 instance;
    private Singleton2() {}
    public static Singleton2 getInstance() {
        if (instance == null) {
            synchronized (Singleton2.class) {
                if (instance == null) {
                    instance = new Singleton2();
                }
            }
        }
        return instance;
    }
}
```

**枚举（推荐之一）**

```java
public enum Singleton3 {
    INSTANCE;
}
```

**静态内部类**

```java
public class Singleton4 {
    private Singleton4() {}
    private static class Holder {
        static final Singleton4 INSTANCE = new Singleton4();
    }
    public static Singleton4 getInstance() { return Holder.INSTANCE; }
}
```

## 面试要点

- 目的：唯一实例 + 全局访问
- 线程安全：饿汉、枚举、静态内部类较省心；懒汉需 DCL + `volatile`
- 枚举单例可防反射与反序列化多次实例化
