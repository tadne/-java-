# 实体类属性名和表中字段名不匹配

列名与属性名不一致时，自动映射失败，需显式指定映射规则。

## 解决方案

| 方式 | 说明 |
|------|------|
| **resultMap** | XML 中逐字段映射，最灵活 |
| **SQL 别名** | `user_id AS id`，配合 `resultType` 自动映射 |
| **全局驼峰** | `map-underscore-to-camel-case: true`（下划线→驼峰） |
| **@Results 注解** | 注解项目省略 XML |

## 1. resultMap

```xml
<resultMap id="userMap" type="User">
  <result column="user_id" property="id"/>
  <result column="user_name" property="name"/>
</resultMap>
<select id="selectById" resultMap="userMap">
  SELECT user_id, user_name FROM user WHERE user_id = #{id}
</select>
```

## 2. 别名

```xml
<select id="selectById" resultType="User">
  SELECT user_id AS id, user_name AS name FROM user WHERE user_id = #{id}
</select>
```

## 3. 驼峰（Boot 常用）

```yaml
mybatis.configuration.map-underscore-to-camel-case: true
```

`user_name` → `userName` 自动映射。

## 面试要点

- 命名规范一致时开驼峰最省事。
- 不规则命名用 resultMap 或别名。
- `@Column` 属于 JPA，不是 MyBatis 标准注解。

## 面试一句话

> 字段名不匹配可用 resultMap、SQL 别名或开启下划线转驼峰，复杂映射优先 resultMap。
