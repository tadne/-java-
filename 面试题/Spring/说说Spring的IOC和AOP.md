# 说说Spring的IOC和AOP

## IoC（控制反转）

**含义**：对象的创建、装配、生命周期由 **Spring 容器**负责，而不是在业务代码里 `new`。

| 概念 | 说明 |
|------|------|
| DI（依赖注入） | IoC 的实现手段：构造器、setter、字段注入 |
| 好处 | 解耦、易测试、易替换实现（配合接口） |

**实现方式**：XML、`@Configuration` + `@Bean`、组件扫描（`@Component` 等）。

## AOP（面向切面编程）

**含义**：把日志、事务、安全等**横切关注点**从业务中剥离，通过切面统一织入。

| 概念 | 说明 |
|------|------|
| 切点 Pointcut | 拦截哪些方法 |
| 通知 Advice | 何时做什么（@Before、@Around 等） |
| 织入 | 运行时动态代理（JDK / CGLIB） |

## IoC 与 AOP 的关系

```
IoC 容器管理 Bean → 对需要增强的 Bean 创建代理 → 调用时执行切面逻辑
```

- `@Transactional`、`@Cacheable` 等注解都依赖 AOP + IoC 协同。

## 对比小结

| | IoC | AOP |
|---|-----|-----|
| 解决的问题 | 对象创建与依赖管理 | 横切逻辑复用 |
| 核心 | 容器、Bean、注入 | 切面、代理、切点 |

## 面试要点

- IoC 是思想，DI 是做法；容器是 `ApplicationContext`。
- AOP 基于代理，只支持方法级连接点（Spring AOP）。
- 两者结合支撑 Spring 声明式事务等特性。

## 面试一句话

> IoC 把对象交给容器创建和注入实现解耦，AOP 用动态代理把横切逻辑织入方法而不改业务代码。
