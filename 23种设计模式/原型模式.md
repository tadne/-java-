# 原型模式

## 思想

通过**复制（克隆）**已有实例来创建新对象，而不是 `new`，避免重复初始化成本。

## 适用场景

- 对象创建成本高（深拷贝数据库对象、复杂配置）
- 需要大量相似对象，仅部分属性不同
- 不想依赖具体类名创建

## 实现方式

| 方式 | 说明 |
|------|------|
| `Cloneable` + `clone()` | 浅拷贝默认；深拷贝需重写 |
| 序列化/反序列化 | 实现深拷贝的笨办法 |
| 拷贝构造函数 | 显式控制拷贝逻辑，更清晰 |

## 代码示例

```java
public class Prototype implements Cloneable {
    private List<String> tags;
    @Override
    public Prototype clone() {
        try {
            Prototype copy = (Prototype) super.clone(); // 浅拷贝
            copy.tags = new ArrayList<>(this.tags);     // 深拷贝引用类型
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new AssertionError(e);
        }
    }
}
```

## 浅拷贝 vs 深拷贝

| | 浅拷贝 | 深拷贝 |
|---|--------|--------|
| 基本类型 | 复制值 | 复制值 |
| 引用类型 | **共享同一对象** | 递归复制新对象 |

## 典型应用

- Java `Object.clone()`（实际项目更常用拷贝构造或 MapStruct）
- Spring `@Scope("prototype")`：每次 getBean 新实例（概念类似，非 clone API）

## 面试一句话

> 原型通过克隆创建对象；注意浅拷贝与深拷贝，引用字段要手动复制。
