# 解释器模式

## 思想

给定一种语言，定义其**文法表示**，并定义一个解释器，用**解释**的方式表示句子。

## 结构

- **AbstractExpression**：声明 `interpret(Context)`
- **TerminalExpression**：终结符表达式
- **NonterminalExpression**：非终结符，组合子表达式
- **Context**：全局/共享信息

## 简单示例

```java
interface Expression {
    boolean interpret(String context);
}

class Terminal implements Expression {
    private String data;
    Terminal(String d) { data = d; }
    public boolean interpret(String ctx) { return ctx.contains(data); }
}

class Or implements Expression {
    private Expression a, b;
    Or(Expression a, Expression b) { this.a = a; this.b = b; }
    public boolean interpret(String ctx) {
        return a.interpret(ctx) || b.interpret(ctx);
    }
}
```

## 典型应用

- 正则表达式引擎
- SQL 解析、EL 表达式（SpEL）
- 简单规则引擎

## 注意

- 适合**简单、稳定**的语法；复杂语法用解析器生成器（ANTLR）更合适
- 类数量随文法规则增长，复杂系统慎用

## 面试一句话

> 解释器为简单语言定义文法类并递归 interpret；复杂语法用 ANTLR 等工具，不必手写解释器树。
