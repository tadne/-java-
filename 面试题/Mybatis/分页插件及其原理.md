# 分页插件及其原理

MyBatis **本身不提供**物理分页，需自行 `LIMIT` 或使用插件。

## 三种方式对比

| 方式 | 是否物理分页 | 说明 |
|------|--------------|------|
| **LIMIT 手写** | ✅ | SQL 中加 `LIMIT offset, size` |
| **RowBounds** | ❌ 逻辑分页 | 内存中截取，数据量大性能差 |
| **PageHelper** | ✅ | 拦截 SQL 改写，生产常用 |

## PageHelper 使用（Boot）

```xml
<dependency>
  <groupId>com.github.pagehelper</groupId>
  <artifactId>pagehelper-spring-boot-starter</artifactId>
</dependency>
```

```java
PageHelper.startPage(1, 10);
List<User> list = userMapper.selectAll();
PageInfo<User> page = new PageInfo<>(list);
```

```yaml
pagehelper:
  helper-dialect: mysql
  reasonable: true
```

## 原理

1. 实现 MyBatis **Interceptor** 接口
2. 拦截 `Executor.query`
3. 用 JSqlParser 解析 SQL，改写为 `SELECT count(*)` + 分页 SQL
4. 将总数与分页结果封装到 `Page` 对象

## RowBounds 说明

```java
sqlSession.selectList("selectAll", null, new RowBounds(0, 10));
```

先查全表再在内存分页，**仅适合小数据量**。

## 面试要点

- 生产推荐 PageHelper 等物理分页。
- 原理：插件 + SQL 解析 + 拦截器链。

## 面试一句话

> MyBatis 分页推荐 PageHelper 拦截器改写 SQL 做物理分页，RowBounds 是内存分页不适合大数据。
