# Java 内存分配

## JVM 内存结构（JDK 8 前后对比）

| JDK 版本 | 内存区域 |
|----------|----------|
| JDK 8 之前 | 栈、堆、方法区（永久代，位于堆中）、本地方法栈、程序计数器 |
| JDK 8 及之后 | 栈、堆、元空间（本地内存）、程序计数器、本地方法栈 |

## 各内存区域说明

| 区域 | 作用 | 与开发关系 |
|------|------|-----------|
| **栈** | 方法运行时使用的内存，如 `main` 方法运行，进入方法栈执行 | 密切相关 |
| **堆** | 存储对象或者数组，`new` 创建的都存储在堆内存 | 密切相关 |
| **方法区** | 存储可运行的 class 文件 | 密切相关 |
| **本地方法栈** | JVM 在使用操作系统功能的时候使用 | 与开发无关 |
| **程序计数器** | 给 CPU 使用，记录当前线程执行位置 | 与开发无关 |

## 核心规则

- **栈**：程序的主入口，开始执行时进栈，执行完毕就出栈。
- **堆**：`new` 创建的都存储在堆中。
- **基本数据类型**：都可以在栈中直接创建，不用 `new`。

## 数组内存分配示例

### 示例 1：new 创建数组

```java
int[] arr = new int[length];
```

- 在堆中创建一个数组空间 `int[]`，分为 `length` 块，每一块中有对应数据。
- 块名也就是索引 0、1、2...，在栈中创建 `arr` 变量对应地址值。
- 使用其中元素的时候就通过地址值在堆中查找对应数据。

> **关键**：只要是 `new` 出来的，一定是在堆里面开辟了一个小空间。如果 `new` 了多次，那么堆里创建多个小空间，每一个小空间中都有各自的数据。

### 示例 2：数组引用赋值

```java
int[] arr1 = {1, 2};
int[] arr2 = arr1;
```

- 这种情况下，`arr2` 不是 `new` 出来的，没有创建新空间。
- 其实是将 `arr1` 的地址赋值给了 `arr2`。
- 所以如果现在通过 `arr2` 修改数组中的属性，那么 `arr1` 中的属性也被修改了。
- 其实就是 `arr2` 和 `arr1` 代表着同一个东西。

> **结论**：两个数组指向同一个小空间时，一个改变，另一个也会改变。

## 面试要点

- **栈**：方法运行时使用的内存，程序主入口，执行完毕出栈
- **堆**：存储对象或数组，`new` 创建的都存在堆中
- **方法区**：存储可运行的 class 文件
- **本地方法栈**：JVM 使用操作系统功能时使用
- **程序计数器**：给 CPU 使用，记录线程执行位置
- **关键规则**：`new` 出来的对象一定在堆中；基本数据类型直接在栈中创建
