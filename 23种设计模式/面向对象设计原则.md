# 面向对象设计原则

面向对象设计中常用 **SOLID + 组合复用 + 迪米特** 等原则，用于降低耦合、提高可维护性。

## 七大原则

| 原则 | 要点 |
|------|------|
| **单一职责（SRP）** | 一个类/方法只做一类变化原因 |
| **开闭（OCP）** | 对扩展开放，对修改关闭 |
| **里氏替换（LSP）** | 子类可替换父类且不改变程序正确性 |
| **依赖倒置（DIP）** | 依赖抽象，不依赖具体实现 |
| **接口隔离（ISP）** | 接口精简，客户端不依赖无用方法 |
| **组合/聚合复用** | 优先组合持有对象，而非盲目继承 |
| **迪米特法则** | 只与直接朋友通信，减少类间耦合 |

## 面试要点

- 记 SOLID 缩写及含义
- 开闭：新增功能尽量加类/实现，少改旧代码
- 里氏：子类加强约束、不破坏父类约定
