# 迭代器模式

## 思想

提供一种方法**顺序访问**聚合对象中的各个元素，而又**不暴露**其内部表示。

## 结构

- **Iterator**：`hasNext()`、`next()`
- **Aggregate**：创建 Iterator
- **ConcreteIterator / ConcreteAggregate**

## JDK 标准

```java
List<String> list = List.of("a", "b");
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
}
// 增强 for 底层即 Iterator
for (String s : list) { }
```

## 典型应用

| API | 说明 |
|-----|------|
| `java.util.Iterator` | 集合遍历 |
| `Iterable` | 可 foreach |
| MyBatis `ResultSet` 游标 | 大结果集迭代 |
| Java 8 `Stream` | 函数式迭代（不同抽象） |

## 内部迭代 vs 外部迭代

| | 外部迭代 | 内部迭代 |
|---|----------|----------|
| 控制 | 客户端控制循环 | 聚合/库控制（如 forEach） |
| 代表 | Iterator | Stream.forEach |

## 面试一句话

> 迭代器统一遍历集合而不暴露内部结构；Java 集合框架 Iterator + Iterable 是标准实现。
