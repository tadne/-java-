# 责任链模式

## 思想

使多个对象都有机会处理请求，将这些对象连成一条**链**，沿链传递请求，直到有对象处理它为止。

## 结构

- **Handler**：定义处理接口，持有 next 引用
- **ConcreteHandler**：处理或转发给 next

## 代码示例

```java
abstract class Handler {
    protected Handler next;
    void setNext(Handler next) { this.next = next; }
    abstract void handle(Request req);
}

class AuthHandler extends Handler {
    void handle(Request req) {
        if (!req.isLogin()) return;
        if (next != null) next.handle(req);
    }
}
```

## 典型应用

| 场景 | 说明 |
|------|------|
| Servlet Filter 链 | 请求经过多个 Filter |
| Spring MVC `HandlerInterceptor` | pre/post/afterCompletion |
| Netty Pipeline | ChannelHandler 链 |
| 审批流 | 多级审批 |

## 纯链 vs 不纯链

- **纯责任链**：只有一个处理器处理，处理完即停
- **不纯链**（更常见）：每个节点都可处理一部分再传递

## 面试一句话

> 责任链把处理器串成链，请求沿链传递；Filter、Interceptor 是 Java Web 中的典型实现。
