# 什么是 JVM，为什么 Java 是跨平台语言

> 标题「跨平台」应为 **跨平台**（Write Once, Run Anywhere）。

## 什么是 JVM

**JVM（Java Virtual Machine）** 是运行 Java 字节码的虚拟机，负责：

| 职责 | 说明 |
|------|------|
| 加载 | 通过类加载器加载 `.class` |
| 验证 | 字节码安全校验 |
| 执行 | 解释执行 + JIT 编译为本地机器码 |
| 内存管理 | 堆、栈、元空间、GC |
| 调度 | 线程与本地方法接口（JNI） |

## Java 程序如何运行

```
.java 源文件
    ↓ javac 编译
.class 字节码（平台无关）
    ↓ 各平台的 JVM 解释 / JIT
本地机器码（平台相关）
```

- **编译一次**：生成统一格式的字节码
- **到处运行**：不同 OS 安装对应 JVM 即可

## 为什么能跨平台

| 层次 | 平台相关？ |
|------|------------|
| Java 源代码 | 否 |
| 字节码 | 否（JVM 规范统一） |
| JVM 实现 | **是**（Windows/Linux/macOS 各有 JVM） |
| 本地机器码 | 是（由该平台的 JVM 生成） |

跨平台的本质是：**把平台差异封装在 JVM 里**，而不是 `.class` 直接在 CPU 上跑。

## JVM、JRE、JDK

| 名称 | 包含 |
|------|------|
| JVM | 虚拟机核心 |
| JRE | JVM + 核心类库（运行环境） |
| JDK | JRE + 开发工具（javac、jar 等） |

## 与其他语言对比

| 语言 | 跨平台方式 |
|------|------------|
| C/C++ | 源码/二进制按平台分别编译 |
| Java | 字节码 + JVM |
| Go | 编译为各平台原生二进制 |
| Python | 解释器跨平台，但依赖解释器环境 |

## 面试一句话

> Java 跨平台靠 JVM：`.class` 字节码统一，由不同操作系统的 JVM 翻译成本地机器码执行。
