# 模板方法模式

## 思想

在父类中定义算法**骨架**（模板方法），将某些步骤延迟到**子类**实现，子类不改变算法结构。

## 结构

- **AbstractClass**：定义 `templateMethod()`，调用抽象/钩子方法
- **ConcreteClass**：实现具体步骤

## 代码示例

```java
abstract class AbstractClass {
    // 模板方法，final 防止子类改流程
    public final void templateMethod() {
        step1();
        if (hook()) step2();
        step3();
    }
    abstract void step1();
    abstract void step3();
    protected boolean hook() { return true; } // 钩子，可选步骤
    private void step2() { /* 默认实现 */ }
}
```

## 典型应用（JDK / 框架）

| 类 | 模板步骤 |
|----|----------|
| `HttpServlet.service()` | doGet / doPost |
| `AbstractList` | 部分方法默认实现 |
| Spring `JdbcTemplate` | 固定获取连接/释放，回调处理 SQL |
| JUC `AbstractQueuedSynchronizer` | tryAcquire 等由子类实现 |

## 钩子方法（Hook）

子类可覆盖钩子改变流程某步是否执行，比纯抽象方法更灵活。

## 面试一句话

> 模板方法在父类固定流程、子类填步骤；HttpServlet、JdbcTemplate 是经典，注意 `final templateMethod`。
