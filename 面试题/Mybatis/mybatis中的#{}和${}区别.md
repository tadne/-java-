# mybatis中的#{}和${}区别

## 对比表

| 对比项 | `#{}` | `${}` |
|--------|-------|--------|
| 处理方式 | **预编译**占位符 `?` | 字符串**拼接**进 SQL |
| 类型 | 自动加引号、类型转换 | 原样替换 |
| SQL 注入 | **防注入**（推荐） | **有风险** |
| 典型用途 | 参数值 | 表名、列名、ORDER BY 字段（需白名单） |

## 原理

- `#{}`：MyBatis 生成 `PreparedStatement`，参数通过占位符绑定。
- `${}`：解析期直接替换到 SQL 字符串，不做预编译。

## 示例

```xml
<!-- 安全：参数值 -->
SELECT * FROM user WHERE id = #{id}

<!-- 动态表名/排序（必须校验输入） -->
SELECT * FROM ${tableName} ORDER BY ${column} ${order}
```

## ORDER BY 注意

- 动态排序字段不能用 `#{}`（会变成带引号的字符串）。
- 使用 `${}` 时必须在代码中**白名单校验**列名，禁止用户直接传入。

## 面试要点

- 值用 `#{}`；动态标识符用 `${}` 且严格校验。
- 永远不要把用户输入未经校验拼进 `${}`。

## 面试一句话

> #{} 预编译防注入用于参数值，${} 字符串拼接用于表名列名等且必须白名单校验。
