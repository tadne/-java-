# 简述 JVM 内存模型（JMM）与运行时数据区

> **易混概念**：JMM（Java Memory Model）是**并发语义规范**；运行时数据区是 **JVM 内存布局**。二者相关但不可混为一谈。

## 一、JMM（Java 内存模型）

定义多线程下**共享变量**的可见性、有序性、原子性规则，核心是 **happens-before**。

### 主内存与工作内存

- 每个线程有**工作内存**（抽象概念，对应 CPU 缓存等）
- 共享变量存于**主内存**
- 线程对变量的操作在工作内存与主内存间同步

### happens-before 常见规则

| 规则 | 含义 |
|------|------|
| 程序顺序 | 同一线程内，前面的操作 hb 后面的（在不影响重排序前提下） |
| 监视器锁 | 解锁 hb 后续对同一把锁的加锁 |
| volatile | 写 volatile hb 后续读该 volatile |
| 线程 start | `start()` hb 该线程内所有操作 |
| 线程 join | 线程内所有操作 hb `join()` 返回 |

### volatile 在 JMM 中的作用

- 保证可见性
- 禁止特定重排序（通过内存屏障实现）
- **不**保证 `i++` 原子性

### synchronized 在 JMM 中的作用

- 释放锁时会把工作内存刷新到主内存
- 获取锁时会从主内存加载最新值

## 二、运行时数据区（JVM 内存结构）

| 区域 | 线程共享 | 作用 | 异常 |
|------|----------|------|------|
| 程序计数器 | 私有 | 当前字节码行号 | — |
| 虚拟机栈 | 私有 | 栈帧、局部变量、操作数栈 | StackOverflow / OOM |
| 本地方法栈 | 私有 | Native 方法 | 同上 |
| 堆 | 共享 | 对象实例、数组 | OOM |
| 方法区（元空间） | 共享 | 类元信息、常量、静态变量 | OOM（元空间） |
| 直接内存 | — | NIO DirectBuffer | OOM |

### 堆的分代（常见）

- **新生代**：Eden + Survivor（From/To）
- **老年代**：长期存活对象
- Minor GC / Major GC / Full GC 与此相关

### 方法区与元空间（易错）

| 版本 | 实现 |
|------|------|
| JDK 7 及以前 | **永久代 PermGen**（在堆内，易 OOM） |
| **JDK 8+** | **元空间 Metaspace**（使用本地内存，默认上限很大） |

> 常见笔误：元空间不是 JDK 9 才引入，而是 **JDK 8** 取代永久代。

### 运行时常量池

- 属于方法区逻辑部分
- 存放字面量、符号引用；`String.intern()` 与之相关

### 直接内存

- 不属于 JVM 规范中的“运行时数据区”，但常与 NIO 一起考
- 由 `DirectByteBuffer` 等使用，受 `-XX:MaxDirectMemorySize` 限制

---

## 三、JMM 与运行时数据区的关系

```
┌─────────────────────────────────────────┐
│  JMM：线程如何看到共享变量的读写顺序      │
│  （规范层，happens-before、volatile）    │
└─────────────────────────────────────────┘
                    ↓ 实现依赖
┌─────────────────────────────────────────┐
│  运行时数据区：堆、栈、元空间等物理划分   │
│  内存屏障、锁、volatile 由 CPU/JVM 实现  │
└─────────────────────────────────────────┘
```

## 面试一句话

> JMM 管多线程可见性与有序性；运行时数据区管 JVM 把内存分成堆、栈、元空间等。**元空间从 JDK 8 开始替代永久代。**
