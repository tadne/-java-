# 策略模式

## 思想

定义一系列**算法**，把它们**封装**起来，并使它们可以**相互替换**；算法变化独立于使用算法的客户端。

## 结构

- **Strategy**：策略接口
- **ConcreteStrategy**：具体算法
- **Context**：持有 Strategy，可运行时切换

## 代码示例

```java
interface PayStrategy {
    void pay(int amount);
}

class AliPay implements PayStrategy {
    public void pay(int amount) { /* ... */ }
}

class PayContext {
    private PayStrategy strategy;
    void setStrategy(PayStrategy s) { this.strategy = s; }
    void checkout(int amount) { strategy.pay(amount); }
}
```

## 典型应用

| 场景 | 说明 |
|------|------|
| `Comparator` | 多种排序策略 |
| Spring 中多种 `Resource` 加载策略 | classpath/file/url |
| 促销规则、路由算法 | 运行时切换 |
| Java 8 Lambda | 函数作为策略传入 |

## 消除 if-else

```java
Map<String, PayStrategy> map = Map.of(
    "ali", new AliPay(),
    "wx", new WxPay()
);
map.get(type).pay(amount);
```

## 面试一句话

> 策略把算法封装成可替换类；Comparator、Lambda 都是策略思想，消除大量 if-else。
